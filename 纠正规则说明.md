# 语音转文字 - 自定义纠正规则说明

## 功能介绍

该程序通过以下方式提高识别准确度：

1. **内置纠正规则** - 处理常见的发音错误和口头禅
2. **自定义纠正规则** - 支持添加你特定的识别错误纠正

## 快速开始

### GUI模式（默认）
```bash
python speech_to_text_unified.py
```

### CLI模式
```bash
python speech_to_text_unified.py --cli
```

## 自定义纠正规则

### 1. 编辑 `corrections.json` 文件

该文件位于程序同一目录下，格式为JSON：

```json
{
  "识别错误的词": "正确的词",
  "你好吗": "你好",
  "怎么样": "怎样"
}
```

### 2. 添加你的纠正规则

根据你的发音习惯和识别错误，添加对应的纠正：

```json
{
  "普通话不标准导致的识别错误": "正确的词",
  "南方口音相关": "标准词",
  "特殊词汇": "规范词汇"
}
```

### 3. 指定自定义规则文件

```bash
python speech_to_text_unified.py --corrections /path/to/your/corrections.json
```

## 常见的识别错误类型

### 1. 发音不标准
- 南北方口音差异（如：零/幺、的/地/得等）
- 卷舌音问题（如：l/n 混淆）
- 平翘舌不分

### 2. 词尾略读
- "了"、"吗"、"呢"等词尾省略或错误
- 量词错误（一个→1个）

### 3. 口头禅
- "呃"、"嗯"、"啊"等语气词
- 重复字词（"好好好"→"好"）

## 建议的纠正方案

### 第一步：记录常见错误
在使用过程中，记下程序识别错误的词汇。

### 第二步：分析错误模式
- 是发音问题还是识别引擎问题？
- 是否有规律性？

### 第三步：更新 `corrections.json`
```json
{
  "你特定的发音错误": "应该是",
  "你另一个识别错误": "正确词汇"
}
```

## 示例

假设你说"西" (xi)，但识别为"吸"，可以添加：
```json
{
  "吸": "西"
}
```

假设你说"我要去"，识别为"我要去去"，可以添加：
```json
{
  "去去": "去"
}
```

## 高级功能

### 标点词汇
你可以直接说标点符号的名称，会自动转换：
- 说"句号" → 。
- 说"逗号" → ，
- 说"问号" → ？
- 说"感叹号" → ！

### 智能断句
程序会自动在合适的位置添加标点符号，如：
- 识别到"然后"、"所以"等词后自动添加逗号
- 识别到"了"、"吗"等词尾自动添加句号

## 注意事项

1. **JSON格式** - 确保 `corrections.json` 是有效的JSON格式
2. **编码** - 文件必须以UTF-8编码保存
3. **优先级** - 自定义规则优先于内置规则
4. **顺序** - 较长的词语纠正规则应放在前面，避免部分匹配问题

## 故障排除

### 自定义规则没有生效
1. 检查JSON文件格式是否正确（可用在线JSON验证器）
2. 确保文件是UTF-8编码
3. 重启程序

### 仍有识别错误
1. 检查发音是否够清晰
2. 考虑说话速度（过快可能导致识别错误）
3. 查看麦克风设置是否正确
4. 考虑在安静的环境中使用
